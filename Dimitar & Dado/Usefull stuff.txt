	* vo git bash
		1) izbiras folder na laptop prajs git init --> go inicijalizira folderot deka e git(ovaj cekor mozis da go skoknis ako sakas)
		2) git clone https://github.com/*****	VASETO	******/FIKT1.git --> praj clone na repository od github kaj vas lokalno
		3) vnesvas username i pass 
		4) Slednite cekori mozite preku browser da gi naprajte(imate sliki postaveno na git) ali visoko preporacvam preku komandna linija
		5) git remote add upstream repo_you_have_cloned(t.e. moeto repo)
		6) git remote -v 
		7) posle odis na git kaj tebe new pull request 
		8) posle create pull request
		9) treba da stigni request kaj mene
		10) git status --> gi prikazva site promeni vo folderot koj sto se raboti
		11) git pull --> gi zema site promeni na odreden file(primer ako rabotite na ist file poke luge vo primerot hello.txt gi zemate promenite na drugiot i posle vrz niv editirate)
		12) git add . --> gi dodadva site files so se editirani, gi dodava vo lokalnoto repository
		13) git commit -m "First commit on github(Vasa message tuka)" --> gi dodava vo stage faza 
		14) git push origin master --> gi dodava na server(github) repository

Sto e sprint?
	* Sprint se definira na 1/2/3 nedeli vo zavisnost od dogovor na timot
	* Najcesto e 2 nedeli, nas ke ni bidi 1 za podobra organizacija
	
Product backlog
	* Tuka ke stojat site tasks za celata aplikacija sto postojat
Spring backlog
	* Tuka ke stojat site tasks sto ke treba da se naprat za eden sprint(edna nedela)
In progress
	* Tuka ke stojat site tasks sto momentalno se rabotat i ke treba da se dodavate kako sto vi pokazvase dadobt
Code review
	* Odkako site sto ke se zavrsi taskot prajte push i tuka nie prajme code review i posle komentirame koj del od kodot e dobar koj e los
Testing
	* Ovaja faza e ko ke ja imame napraeno aplikacijata i ko ke mozi veke nesto realno da se testira
Done
	* Tuka vlegvat site zavrseni tasks za odreden sprint
